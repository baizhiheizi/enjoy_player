# ASR transcript generation fixtures

Local-only audio / video fixtures used by the `asr-transcript-generation`
feature tests. **No real media is committed** — these paths are referenced
by tests that mock FFmpeg / `AsrService` directly.

## Expected fixtures (developer-supplied, gitignored)

| Path | Purpose |
|---|---|
| `en_5min.m4a` | ~5-minute English audio for `quickstart.md` Scenarios A, D, E, F. |
| `en_5min.mp4` | Same audio in a video container for Scenarios B, I. |
| `en_45min.mp4` | Long-media fixture for QR-008 / FR-008 confirmation dialog. |

## Test-only stubs (safe to commit)

- `silence.wav` — a 1-second silent WAV generated by the unit tests via
  `dart:typed_data`. Used for `asr_audio_extractor_test.dart` and
  `asr_generation_controller_test.dart` to avoid touching real audio.

## Why no committed audio

Real audio fixtures are large and often licensed. The unit tests mock
`AsrAudioExtractor` and `AsrService` so they don't need the file bytes.
`integration_test/asr_generate_transcript_test.dart` is the only place
that needs real media, and it runs against a developer-supplied file
via a path environment variable — see `docs/features/asr.md` once
published.
